在 Waline 中使用 Neon 数据库 (Vercel 部署)

注意：根据 Vercel 的部署策略，必须为 Waline 服务端绑定自定义域名才能正常访问。使用免费的 *.vercel.app 域名可能会导致访问异常。

本指南基于 Waline 官方文档进行整理，官网提供了非常详细的说明。

第一步：部署 Waline 服务端到 Vercel

  1. 进入 Waline 快速开始页面：访问 https://waline.js.org/guide/get-started/。
  2. 通过 Vercel 部署：
    • 点击页面上的 “Deploy to Vercel” 按钮。
    • 如果你未登录，Vercel 会引导你使用 GitHub 账户登录或注册。
  3. 创建 Vercel 项目：
    • 登录后，为你的项目起一个名称，然后点击 “Create”。
    • Vercel 会自动基于 Waline 模板创建仓库并开始部署。稍等片刻，部署成功后会出现庆祝页面。

第二步：配置 Neon 数据库

  1. 进入数据库设置：
    • 在 Vercel 项目控制台，点击顶部导航栏的 “Storage”。
    • 在存储页面，点击 “Create Database”。
  2. 选择并创建 Neon 数据库：
    • 在数据库服务商列表中，选择 “Neon”，然后点击 “Continue”。
    • 系统会提示你创建或关联 Neon 账户。点击 “Accept and Create” 接受并创建。
    • 后续步骤（选择地区、套餐、数据库名称）可以保持默认，一路点击 “Continue” 完成创建。
  3. 初始化 Waline 数据库表：
    • 数据库创建后，在 Vercel 的 Storage 页面点击该数据库，选择 “Open in Neon”，跳转到 Neon 控制台。
    • 在 Neon 控制台左侧，选择 “SQL Editor”。
    • 从 Waline 官方仓库获取 https://github.com/walinejs/waline/blob/main/assets/waline.pgsql 文件中的 SQL 语句，粘贴到编辑器中，点击 “Run” 执行，以创建所需的数据表。
  4. 重新部署 Vercel 应用：
    • 回到 Vercel 项目控制台的 “Deployments” 页面。
    • 找到最新的那次部署，点击右侧的 “Redeploy” 按钮，让刚刚配置的数据库生效。

第三步：（关键）绑定自定义域名

  1. 添加域名：
    • 部署完成后，进入 Vercel 项目的 “Settings” -> “Domains”。
    • 在输入框中填入你已拥有的自定义域名（例如 comment.yourdomain.com），点击 “Add”。
  2. 配置 DNS 解析：
    • Vercel 会显示需要你添加到域名注册商处的 DNS 记录（通常是 CNAME 记录）。
    • 前往你的域名管理后台，按照 Vercel 的提示添加这条 CNAME 记录。
  3. 等待生效：
    • DNS 记录生效可能需要几分钟到几小时。生效后，即可通过你的自定义域名访问 Waline 服务端。

第四步：在配置文件中引入 Waline

第五步：初始化管理员

  1. 访问你的 Waline 管理界面：https://你的自定义域名/ui/register。
  2. 第一个在此页面注册的用户将被自动设置为管理员。

关于免费域名

为了完成第三步的“绑定自定义域名”，你需要拥有一个域名。

  • 免费域名来源：你可以寻找提供免费域名的服务。例如，B站 UP主“Porter科技迷”发布的视频 《免费域名合集 | 支持托管Cloudflare | 白嫖域名 | 免实名无门槛 | 从注册域名到托管cloudflare全过程演示》 中，详细介绍了多个当前可用的免费域名注册渠道（如 dnshe、digitalplat 等）、它们的优缺点以及完整的使用流程。
  • 具体使用参考：视频链接：https://b23.tv/w8YB3bi
