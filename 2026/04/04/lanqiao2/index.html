虚拟滚动列表

虚拟列表其实是按需显示的一种实现，即只对可见区域进行渲染，对非可见区域中的数据不渲染或部分渲染的技术，从而达到极高的渲染性能。 比较适用于数目较多但体量不大的数据渲染应用场景，后端接口可一次性返回条数较多的数据，避免向服务器频繁地发送 http 请求去多次获取，比如新闻标题列表。

假设接口一次性返回了 1 万条记录，我们屏幕的可见区域的高度为 500px，而列表项的高度为 50px，则此时我们在屏幕中最多只能看到 10 个列表项，在不考虑分页的情况下，如果我们一次性渲染 1 万个 DOM 节点，性能开销会非常大。 如果在首次渲染的时候，我们只需加载 10 条，渲染就会变得非常快：

虚拟列表的实现，实际上就是在首屏加载的时候，只加载可视区域内需要的列表项，当滚动发生时，通过动态计算获得可视区域内的列表项，并将非可视区域内已存在的列表项删除：

计算当前可视区域起始数据索引（startIndex）。
计算当前可视区域结束数据索引（endIndex）。
计算当前可视区域的数据，并渲染到页面中。
计算（startIndex）对应的数据在整个列表中的偏移位置（startOffset）并设置到列表上。

scrollEvent(e) {
this.scrollTop=e.target.scrollTop
return Math.floor(this.scrollTop/this.itemHeight)

e：事件对象
e.target：触发滚动事件的 DOM 元素
scrollTop：元素顶部被隐藏的像素数（从顶部滚动了多少距离）

    通过 slice方法截取需要显示的数据
  2. 滚动事件 scrollEvent： 获取滚动容器的 scrollTop值 根据滚动距离和元素高度计算当前应该显示的第几个元素 更新 start索引，触发计算属性重新计算
  3. 数据加载： 使用 axios获取数据 计算总高度：列表长度 × 每个元素高度 添加错误处理，在请求失败时使用模拟数据
  4. 虚拟滚动原理： 滚动容器：固定高度，设置 overflow-y: auto 占位元素：scroll-container用于撑开滚动区域 虚拟列表：list使用 transform: translateY进行定位 数据动态加载：只渲染可视区域及缓冲区内的元素
  5. 优化点： 使用 transform代替 top进行定位，性能更好 设置 buffer缓冲区，防止滚动时出现白屏 通/过计算属性自动响应数据变化 支持异步数据加载
    虚拟列表工作原理总结：
  6. 数据分片渲染：不一次性渲染所有数据，只渲染可视区域的部分
  7. 滚动位置计算：通过 scrollTop和 itemHeight计算当前应该显示哪些数据
  8. 缓冲区优化：在可视区域外额外渲染一些元素，防止滚动时出现空白
  9. CSS变换定位：通过 translateY移动内容容器，模拟整个列表的滚动效果
  10. 固定高度假设：需要知道每个列表项的精确高度，这是实现简单虚拟列表的前提
    性能优势：
  • 无论数据量多大（1万、10万条），DOM元素数量基本恒定
  • 内存占用低，滚动流畅
  • 适合大数据列表展示场景

防抖 (Debounce)

function debounce(fn, delay) {
let timer = null;
return function(…args){
clearTimeout(timer)
const context=this
timer=setTimeout(()=>{
fn.apply(this,args)
},delay)
}
}

// 使用示例
const handleScroll = debounce((e) => {
console.log(‘Scroll event handled’, e.target.scrollTop);
}, 200);

element.addEventListener(‘scroll’, handleScroll);

节流 (Throttle)

// 时间戳版本（立即执行）
function throttle(fn, delay) {
let lastTime = 0;

return function(…args) {
const now = Date.now();
const context = this;

if (now - lastTime >= delay) {
  fn.apply(context, args);
  lastTime = now;
}

};
}

// 定时器版本（延迟执行）
return function(…args) {
const context = this; // 保存当前 this 上下文

if (!timer) { // 如果没有正在等待的定时器
fn.apply(context, args); // 立即执行函数

timer = setTimeout(() => {  // 设置一个定时器
  timer = null;  // 延迟时间结束后，重置定时器标识
  
  // 检查是否有保存的最后一次调用参数
  if (lastArgs) {
    // 执行最后一次保存的调用
    fn.apply(lastContext, lastArgs);
    lastArgs = null;     // 清空保存的参数
    lastContext = null;  // 清空保存的上下文
  }
}, delay);

} else { // 如果已经有定时器在等待
// 保存最后一次调用的参数和上下文
lastArgs = args;
lastContext = context;
}
};

// 使用示例
const handleResize = throttle((e) => {
console.log(‘Window resized’, window.innerWidth);
}, 100);

window.addEventListener(‘resize’, handleResize);
防抖：事件触发后等待一段时间执行，期间重复触发会重置等待
节流：在指定时间间隔内只执行一次
适用场景：
防抖：搜索框输入、窗口 resize 结束后的处理
节流：scroll 事件、鼠标移动、窗口 resize 过程中的处理
