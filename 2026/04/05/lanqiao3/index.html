搜索处理，this指向

为输入框（即 this.inputEl）绑定 input 事件，当输入框的值发生变化时，调用已经提供的 handleInput 方法进行搜索处理，注意 handleInput 方法调用时的 this 指向应为 search 对象本身。

箭头函数没有自己的 this，它会继承外层作用域的 this值
所以这里：
事件监听器的回调函数是箭头函数
箭头函数继承了外层 handle()方法中的 this
因此箭头函数中的 this也指向 search 对象

handle() {
// 这里的 this 指向 search 对象
// 因为 handle() 是作为 search 对象的方法被调用的
console.log(this); // 输出：search 对象
console.log(this.inputEl); // 输出：输入框元素
}
this指向
handle()方法内部 search 对象
作为对象方法调用，箭头函数回调中，search 对象

这就是为什么这里使用箭头函数很重要，它保证了在事件处理程序中仍然能访问到 search对象，从而可以调用 this.handleInput()和 this.render()等方法。

const words = [
{

“meaning”: [
“v. 抛弃，舍弃; 中止，放弃; 放纵”,
“n. 放任，放纵”
]

word 表示单词。
meaning 表示词义。
补全 waitReview 函数，当点击待复习按钮（id=”wait_btn”）时会调用此函数，函数功能为跳过当前单词，渲染下一个单词。如果当前单词为 words 数据中的最后一个单词，则跳到第一个单词。变量 index 为当前渲染的单词在 words 数组中的序号值（即索引+1）。例如第一个单词为 words[index-1]。

补全 completeReview 函数，当点击已复习按钮（id=”complete_btn”）时会调用此函数，函数功能为从 words 中删除当前单词，表示该单词已复习完毕，并渲染下一个单词。如果当前单词为 words 数据中的最后一个单词，则跳到第一个单词；如果单词全部复习完毕时，单词元素（id=”word”）的显示内容为：

今日复习已完成

。

提示：waitReview 函数和补全 completeReview 函数渲染单词调用已提供的 renderData 函数，并将要渲染单词的序号值传递过去。例如渲染第一个单词调用为renderData(1)，渲染words[0] 单词为 abandon。

洗牌算法

从数组末尾开始，随机选择一个位置（包括当前位置）的元素与当前元素交换，逐步向前遍历。
时间复杂度：O(n)，空间复杂度：O(1)（原地打乱）

  1. 多媒体与娱乐
    音乐播放器：随机播放歌单，避免重复模式。
    const playlist = [“歌曲A”, “歌曲B”, “歌曲C”, “歌曲D”];
    const shuffledPlaylist = fisherYatesShuffle([…playlist]);
    // 随机播放顺序
    视频/短片推荐：打乱推荐内容顺序，增加多样性。
    播客节目：随机排列节目列表，提供新鲜感。
  2. 游戏开发
    扑克牌/卡牌游戏：洗牌是核心操作。
    // 斗地主发牌前洗牌
    const deck = […Array(54).keys()]; // 54张牌（含大小王）
    const shuffled = fisherYatesShuffle(deck);
    const player1 = shuffled.slice(0, 17); // 发牌

str.match(reg)

const result = string.match(regex)
匹配成功（返回数组）
const match = url.match(/[?&]wd=([^&]*))
// match[0]: 整个匹配的字符串（如 “?wd=JavaScript”）
// match[1]: 第一个括号捕获的内容（如 “JavaScript”）
// match[2]: 第二个括号捕获的内容（如果有更多括号）

( ) ← 捕获组开始
[^&] ← 匹配”除了&以外的任何字符”
* ← 匹配0次或多次
match[1]：数组索引，获取括号捕获的第1个内容
([^&]*)：括号捕获，^在里面是”非”， *是”多次”
$&：在替换时代表”整个匹配的文本”
const escaped=searchkey.replace(/[.*+?^${}()|[]\]/g,’\$&’)
bd.innerHTML.replace(reg,’$&‘)
**$&**是 JavaScript replace()方法中的特殊替换模式，表示匹配到的整个文本
替换字符串需要用引号包裹,所以 ‘$&‘是正确的

str.replace()/str.replaceAll()

str.replace()
只替换第一个匹配,可控制替换数量（通过 g标志）
str.replaceAll()
替换所有匹配,必须包含 g标志
str.replace(/apple/g, “orange”);
str.replaceAll(/apple/g, “orange”);
// 使用 $& 表示匹配的文本
str.replaceAll(/\d+/g, ‘$&‘);

str.replace(pattern, replacement)

  1. pattern- 要查找的模式
    可以是字符串（只替换第一个匹配项）
    也可以是正则表达式（可替换所有匹配项）

  2. replacement- 替换的内容
    可以是字符串
    也可以是函数（用于更复杂的替换逻辑）

  3. 交换顺序
    let str = “张三,李四”;
    str.replace(/(.+),(.+)/, ‘$2,$1’); // “李四,张三”
    (.+) , (.+)
    ↑ ↑ ↑ ↑ ↑
    │ │ │ │ └── 匹配任意字符1次或多次
    │ │ │ └─── 第2个捕获分组开始
    │ │ └────── 匹配逗号字符
    │ └────── 匹配任意字符1次或多次
    └────── 第1个捕获分组开始

  4. 重新格式化
    let phone = “13800138000”;
    phone.replace(/(\d{3})(\d{4})(\d{4})/, ‘$1****$3’);
    // 结果：”138****8000” (隐藏中间4位)

  5. 格式化日期
    let date = “2023-12-25”;
    // 美国格式：月/日/年
    date.replace(/(\d{4})-(\d{2})-(\d{2})/, ‘$2/$3/$1’); // “12/25/2023”

// 欧洲格式：日/月/年
date.replace(/(\d{4})-(\d{2})-(\d{2})/, ‘$3/$2/$1’); // “25/12/2023”

map()

map()就是把数组的每个元素”映射”成新值，组成新数组。

  1. 基本转换（最常用）
    // 数组每个元素乘以2
    const nums = [1, 2, 3]
    const doubled = nums.map(n => n * 2) // [2, 4, 6]

// 提取对象属性
const users = [{name: ‘张三’, age: 20}, {name: ‘李四’, age: 25}]
const names = users.map(user => user.name) // [‘张三’, ‘李四’]
2. 修改对象结构
// 重新组织对象
const products = [
{id: 1, title: ‘手机’, price: 1999},
{id: 2, title: ‘电脑’, price: 5999}
]

const productList = products.map(p => ({
productId: p.id,
productName: p.title,
price: p.price + ‘元’
}))
// [{productId:1, productName:’手机’, price:’1999元’}, …]
3. 结合解构

]

// 添加索引

// 数据处理管道

将平铺的结构转化为树状结构：

[
{
id: “51”, // 区域 id
name: “四川省”, // 区域名字
pid: “0”, // 区域的父级区域 id
},
{
id: “5101”,
name: “成都市”,
pid: “51”, // 成都的父级是四川省，所以 pid 是 51
},
// …
[
{
id: “51”, // 地址 id
name: “四川省”, // 地址名
pid: “0”, // 该地址的父节点 id
children: [
{
id: “5101”,
name: “成都市”,
pid: “51”,
children: [
{
id: “510101”,
name: “市辖区”,
pid: “5101”,
children: [], // 如果该区域节点没有子集，children 则为空数组！！！
},
// …
],
},
// …
],
},
// …
];

接收平铺的区域信息数组，并将其转化为树状结构，最终数据结构如上面介绍中所示（树状结构，且对于没有子集的叶子节点，其 children 属性设置为空数组）。
并且还接收一个 rootId，用于标识返回哪一个区域下面的所有节点。默认为 0 表示返回所有省份信息（在我们的示例数据中，所有省份的 pid 都为 0）。

Map()

result只存储根节点的直接子节点
节点1: 是根节点的直接子节点 → 进入 result
节点2: 是节点1的子节点 → 进入节点1的 children
节点3: 是节点1的子节点 → 进入节点1的 children
节点4: 是节点2的子节点 → 进入节点2的 children
节点5: 是节点2的子节点 → 进入节点2的 children
nodeMap的作用是：
快速查找：通过 ID 快速找到节点引用（O(1) 复杂度）
引用共享：确保父子节点引用的是同一个对象
简化逻辑：避免多层循环和递归
性能优化：从 O(n²) 优化到 O(n)
虽然看起来多了一个数据结构，但这是用空间换时间的典型优化策略，对于大数据量的树形结构转换，这种优化是必要的

proxy 代理

在文件 age.js 中，有一个 person 对象，当它的 age 属性值改变时需要对新值做判断，将其永远控制在 0 - 150 之间。具体需求如下：
如果新属性值在 0 -150 之间（包含 0 和 150），则直接更新。
如果新属性值小于 0，则属性值更新为 0。
如果新属性值大于 150，则属性值更新为 150。
