一、Promise 是什么？

Promise 是 JavaScript 中处理异步操作的对象，它代表一个尚未完成但预期将来完成的操作。
记住 Promise 的核心：将异步操作封装成对象，用 .then()和 .catch()处理结果，避免回调地狱。

二、Promise 的三种状态

  1. Pending（进行中）
    const pendingPromise = new Promise(() => {});
    console.log(pendingPromise); // Promise {}

  2. Fulfilled（已完成）
    const resolvedPromise = Promise.resolve(“成功”);
    console.log(resolvedPromise); // Promise {: “成功”}

  3. Rejected（已失败）
    const rejectedPromise = Promise.reject(“失败”);
    console.log(rejectedPromise); // Promise {: “失败”}

九、常见陷阱

  1. 陷阱1：忘记返回
    // ❌ 错误
    somePromise()
    .then(value => {
    anotherPromise(value); // 忘记 return！
    })
    .then(result => {
    // result 是 undefined
    });

// ✅ 正确
somePromise()
.then(value => {
return anotherPromise(value); // 记住 return！
});
2. 陷阱2：错误处理不当
// ❌ 错误 - 同步错误不会捕获
new Promise((resolve, reject) => {
throw new Error(“错误”); // 需要在 try-catch 中
});

// ✅ 正确
new Promise((resolve, reject) => {
try {
throw new Error(“错误”);
} catch (error) {
reject(error);
}
});

某电商平台接收到一笔订单，其中包含多个商品，每个商品需要使用不同的物流配送服务。需要对订单进行拆分，并使用相应的物流配送服务进行配送，需要获取到不同物流配送的结果之后再更新订单状态。
找到 js/index.js 文件中的 myPromiseAll 函数，在不使用 Promise.all 的情况下，完善其中的 TODO 部分，目标如下：
myPromiseAll 接收一个参数，该参数为一个数组，这个数组中可以包含 Promise 对象或普通值（基本数据类型） ，myPromiseAll 返回一个 Promise，解析为结果数组。

  1. 当参数为空数组时，应该解析为空数组。
  2. 当传递数组中的值为 Promise 成功值或者普通值时，正确解析结果数组，参数中的 Promise 成功值应该为解析后的值，如 Promise.resolve(1) 解析后的结果应为 1，普通值仍为值本身。
  3. 当传递数组中的值中包含一个被拒绝的 Promise 时（如 Promise.reject()），应该被参数数组中第一个 reject 终止，并输出该 reject 接收到的错误信息。
