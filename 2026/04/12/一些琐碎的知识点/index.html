for-in 语句

for-in语句是一种严格的迭代语句，用于枚举对象中的非符号键属性，语法如下：
for (const propName in window) {
document.write(propName);}
这个例子使用 for-in 循环显示了 BOM对象 window的所有属性。

for-of 语句

是一种严格的选代语句，用于遍历可送代对象的元素，语法如下：
for (property of expression) statement 下面是示例：
for (const el of [2, 4, 6, 8]){
document.write(el);}
在这个例子中，我们使用for-of语句显示了一个包含4个元素的数组中的所有元素。会持续到将所有元素都选代完。与for循环一样，这里控制语句中的const 也不是必需的。这个局部变量不被修改，推荐使用const。

for-of 循环会按照可选代对象的next.()方法产生值的顺序送代元素。关于可选代对象，本机第7章详细介绍。
如果尝试迭代的变量不支持送代，则for-of语句会抛出错误。

全等和不全等

全等和不全等操作符与相等和不相等操作符类似，只不过它们在比较相等时不转换操作数。全等作符由3个等于号(===）表示，只有两个操作数在不转换的前提下相等才返回 true，比如：
let result1 =(“55”== 55)；I/ true,:转换后相等
let result2 = (“55”=== 55)；// false，不相等，因为数据类型不同
在这个例子中，第一个比较使用相等操作符，比较的是字符串”55“和数值

递增/递减操作符

递增和递减操作符直接照搬自C语言，但有两个版本：前缓版和后缀版。顾名思义，于要操作的变量前头，后缀版就是位于要操作的变量后头。前缀递增操作符会给数值加(++)放到变量前头即可：
let age = 29;++age;
在这个例子中，前缀递增操作符把age 的值变成了30（给之前的值29加1)。因如下表达式：
let age = 29;age = age + 1;
前缀递减操作符也类似，只不过是从一个数值减1。使用前缀递减操作符，只放到变量前头即可：
let age = 29;–age;
执行操作后，变量age的值变成了28（从29减1)。

v-if Vs v-show

v-if是“真实的“按条件渲染，因为它确保了在切换时，条件区块内的事件监听器和子组件都会被销毁与重建vif也是惰性的：如果在初次渲染时条件值为 false，则不会做任何事。条件区块只有当条件首次变为true时才被渲染。
相比之下，vshow 简单许多，元素无论初始条件如何，始终会被渲染，只有 CSS display 属性会被切换。
总的来说，v-if有更高的切换开销，而v-show 有更高的初始渲染开销。因此，如果需要频繁切换，则使用v-show 较好；如果在运行时绑定条件很少改变，则v-if 会更合适

nvm

nvm (Node Version Manager) 是一个Nodejs的版本管理工具。
nvm官网：https://nvm.uihtm.coml
nvm的作用：
多版本管理：nvm允许你在同一台设备上安装和管理多个Nodejs版本
版本切换：你可以根据需要轻松切换到特定的Node.js版本，以便在不同项目之间切换
解決兼容性问题：对于依赖不同Nodejs版本的项目，nvm提供了一种简单的方法来管理和切换版本，从而解决兼容性问题
