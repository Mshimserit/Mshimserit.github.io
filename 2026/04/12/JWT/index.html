JWT (JSON Web Token) 详解

📌 什么是JWT？

定义

JWT（JSON Web Token）是一种开放标准（RFC 7519），用于在网络应用环境间安全地传输信息。它采用JSON对象作为载体，并使用数字签名保证信息不被篡改。

主要特点

  • 紧凑：体积小，可通过URL、POST参数或HTTP头发送
  • 自包含：包含所有必要信息，减少数据库查询
  • 可验证：通过签名确保数据完整性
  • 标准化：遵循RFC标准，跨语言支持

🔐 JWT结构

JWT由三部分组成，用点号（.）分隔：

header.payload.signature

    📊 JWT vs Session vs OAuth2

特性 JWT Session OAuth2
状态 无状态 有状态 协议框架
存储 客户端 服务器端 令牌
扩展性 优秀 一般 优秀
适合场景 API认证 传统Web应用 第三方授权
安全性 依赖签名 依赖会话管理 最完整

🚨 常见攻击与防护

1. 令牌窃取

  • 防护：使用HTTPS、短期令牌、令牌绑定

2. 重放攻击

  • 防护：使用jti、nonce、时间戳验证

3. 算法混淆攻击

// ✅ 明确指定算法
jwt.verify(token, secret, { algorithms: [‘HS256’] });
