钱包余额问题

用于在浏览器中管理以太坊账户。为了方便用户使用，小蓝决定将钱包分为两个页面：

  • 存款页面，用于用户存款。
  • 钱包页面，用于查看账户余额。
    但是小蓝遇到了问题，两个页面显示的账户余额不一致。

  1. 找到 index.html 中的 TODO 部分，为项目配置 history 模式路由，浏览器中访问 / 的时候显示 WalletPage 组件，访问 /deposit 的时候显示 DepositPage 组件。
    tips: 目标 1 完成后点击 deposit 按钮或底部的导航的 deposit 导航文字，均会跳转到 DepositPage 存款页面。

  2. 找到 DepositPage.js 中的 TODO 部分，在 DepositPage 页面中的 （id = deposit-balance）元素正确显示钱包余额（store 中的 balance）。

  3. 完善 js/store.js 和 component/DepositPage.js中的 TODO 部分，在 DepositPage 页面中，在输入框（input）输入数字（只考虑正整数），点击 “Deposit” 按钮（button）后，余额 = 现在的余额 + 输入框中输入的金额，在 DepositPage 页面正确显示钱包余额（两个页面的余额相同）。初始余额为 23。
    用户操作流程：

  4. 访问首页 (/) → 显示 WalletPage
    ├─ 显示当前余额: store.balance (23 ETH)
    └─ 点击”Deposit”按钮 → window.router.push(‘/deposit’)

  5. 跳转到存款页 (/deposit) → 显示 DepositPage
    ├─ 输入存款金额 → 保存在 amount
    └─ 点击”Deposit”按钮 → 触发 handleDeposit
    ├─ store.setVal(金额) → balance.value += 金额
    └─ router.push(‘/‘) → 返回首页

  6. 返回首页 → 显示更新后的余额

字符串是不可变的（immutable），所有字符串方法都不会修改原字符串，而是返回一个新的字符串。
slice()方法用于提取字符串的一部分，并返回提取的新字符串，原字符串保持不变。

  1. 完成十种花卉的展示
    所有花卉的信息存放于 js/flowers.json 中，请在 index.html 中使用 axios 发送请求（请求地址写死为 ./js/flowers.json ，以免影响检测通过）获取到花卉信息，将其保存至 js/store.js 的 flowers 数组中，并在 components/FlowerList.js 组件中将所有花卉渲染在页面上（每条花卉信息是一个 div 标签，需要生成对应花卉数量的 div 标签作为子元素插入到 .flower-list 中）。
