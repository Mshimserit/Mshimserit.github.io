评分组件封装

  1. my-rate.vue 组件能够对不同的维度进行评分。

  2. my-rate.vue 组件对外抛出 change 事件，在三项评分均完成后，触发 change 事件，change 事件包含一个参数，用于传递改变后的分数值，其类型是对象，包含以下属性：
    {
    speed: number;
    flavour: number;
    pack: number;
    }
    State = 数据仓库（存什么）
    Getters = 数据计算（怎么算）
    Actions = 业务逻辑（做什么）

    this.ratings​ - 直接访问 store 实例上的 ratings属性
    这是 Pinia 提供的 getter 访问方式
    底层会代理到 this.$state.ratings
    推荐使用这种方式
    state.ratings​ - 通过 state 参数访问
    只能在定义 action 时通过解构参数获得
    不推荐，因为会丢失响应性优化
    在 Pinia 的 actions 中，this 指向的就是当前 store 实例，所以可以直接用 this.ratings[type] 修改 state。
    this.ratings[type] = score
    动态属性名：原函数中的 type是参数，表示动态的属性名，所以必须使用方括号表示法 this.ratings[type]来访问和设置属性。

    简单判断标准：

  3. 只读显示 → 直接 store.xxx

  4. 需要 v-model → 用 computed包装

  5. 要解构多个属性 → 用 storeToRefs
    在你的代码中，两种方式都是合适的，取决于具体的需求。

  6. 使用 storeToRefs（解构响应式）
    //storeToRefs 只能解构 store 的顶级属性
    const { ratings, isCompleted, averageScore } = storeToRefs(ratingStore)

// 使用时要区分：
console.log(ratings.value.speed) // 在 script 中

// 在 template 中（自动解包）
  1. 直接访问 store.ratings.speed
    适用场景：
  • 只读显示，不需要修改
  • 简单的单向数据流
  • 偶尔使用的简单组件
  • 不需要 v-model的场景
    优点：最简单直接，无额外代码
    缺点：模板冗长，双向绑定麻烦
  1. 使用 computed包装（推荐用于表单）
    适用场景：
    -需要 v-model双向绑定
  • 表单控件（input、select、rate等）
  • 组件中频繁读写这个值
  • 想要简洁的模板代码
    优点：支持 v-model，模板简洁
    缺点：需要额外定义 computed
  1. 使用 storeToRefs解构
    适用场景：
  • 需要解构多个 store 属性
  • 想要简化模板中的 ratingStore.前缀
  • 需要在多个地方使用相同属性
    优点：可以一次解构多个属性
    缺点：不支持直接 v-model，仍需要手动绑定

ISBN 转换与生成

  1. 补充 getNumbers 函数，剔除输入参数 str 中除了数字和大写 X 之外的其他字符，将其转换为只有纯数字和大写 X 字母的字符串。
  2. 补充 validISBN10 函数，判断输入参数 isbn 是否是一个有效的 ISBN-10 字符串，并将判断结果（true 或 false）返回。有效的 ISBN-10 判断方法如下：
  • 有效的 ISBN-10 字符串是只有纯数字和大写 X 字母的字符串，其前九位是 0-9 之间的任意数字，最后一位校验位的值取决于前九位数字。
  • 校验位计算方法：用 1-9 这 9 个数依次乘以前面的 9 位数，然后求它们的和除以 11 的余数。如果余数为 10，则校验码用 X 表示，否则，校验码用该余数表示。
  • 以 7-5600-3879-4 为例，它的前 9 位数是 7、5、6、0、0、3、8、7、9，则其校验码的计算过程如下：
    1x7+2x5+3x6+4x0+5x0+6x3+7x8+8x7+9x9

= 7+10+18+0+0+18+56+56+81

= 246

246 % 11 = 4

因此，这个 ISBN-10 字符串的校验码就是4。

  • 以 2-5600-3879-X 为例，它的前 9 位数是 2、5、6、0、0、3、8、7、9，则其校验码的计算过程如下：
    1x2+2x5+3x6+4x0+5x0+6x3+7x8+8x7+9x9

= 2+10+18+0+0+18+56+56+81

= 241

241 % 11 = 10

因此，这个 ISBN-10 字符串的校验码就是X。
3. 补充 ISBN10To13 函数，将输入参数 isbn (一个有效的 ISBN-10 字符串) 转化为对应的 ISBN-13 字符串，并将转化后的字符串返回。转化步骤如下：

  • 将 ISBN-10 字符串的最后一位校验位去掉，剩下前九个数字。
  • 在字符串开头增加 978 三个数字，获得长度为 12 的数字字符串。
  • 计算最后一位校验位。ISBN-13 的校验码计算规则如下：用1分别乘书号的前12位中的奇数位, 用3乘以偶数位，然后求它们的和除以10的余数，最后用10减去这个余数，就得到了校验码。如果余数为0，则校验码为0。
  • 比如，7-5600-3879-4 在 ISBN-13 中，就是 978-7-5600-3879-7。它的校验码计算方法如下：
    9x1+7x3+8x1+7x3+5x1+6x3+0x1+0x3+3x1+8x3+7x1+9x3

= 9+21+8+21+5+18+0+0+3+24+7+27

= 143

143 % 10 = 3

10 - 3 = 7

因此，这个 ISBN-13 字符串的校验码就是7。

Vue 3 中 reactive 与 ref 对比

ref vs reactive

创建方式

• ref: 通过 ref(value) 函数创建

• reactive: 通过 reactive(object) 函数创建

返回值

• ref: 返回一个包含 .value 属性的响应式对象

• reactive: 返回一个响应式 Proxy 对象

适用类型

• ref: 适用于所有类型（基本类型、对象、数组等）

• reactive: 只能用于对象（包括数组、Map、Set 等）

模板自动解包

• ref: 是（在模板中不需要使用 .value 访问）

• reactive: 是（可以直接访问属性）

解构丢失响应性

• ref: 是（需要使用 toRefs 处理才能保持响应性）

• reactive: 是（需要使用 toRefs 处理才能保持响应性）

重新赋值

• ref: 可以直接替换整个值（通过 .value）

• reactive: 不能直接替换整个对象，只能修改对象的属性

watch 监听

• ref: 可直接监听

• reactive: 可直接监听
