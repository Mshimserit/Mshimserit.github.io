查询地址

从树状结构的信息中，遍历找到目标区域的信息，如输入：成都市，返回 [四川省，成都市]

  1. 初始化：把所有顶层区域入栈
  2. while循环（栈不为空）：
    a. 弹出栈顶元素
    b. 检查：是不是要找的区域？
    是 → 返回path（找到啦！）
    否 → 继续
    c. 检查：有没有子节点？
    有 → 把子节点都入栈（记录路径）
    无 → 继续while循环
  3. 栈空了还没找到 → 返回null（不存在）

// 原始写法
mentalMethod(‘峨眉’)(‘武当’)(‘少林’)();

// 分解写法
const step1 = mentalMethod(‘峨眉’); // 返回函数
const step2 = step1(‘武当’); // 返回函数
const step3 = step2(‘少林’); // 返回函数
const result = step3(); // 返回字符串 “战胜峨眉,武当,少林”

mentalMethod 需要返回一个函数，可以一直进行调用，但是最后一次调用不传参。
函数通过以下方式执行，返回结果均为 ‘战胜峨眉,武当,少林’。
mentalMethod(‘峨眉’)(‘武当’)(‘少林’)();
mentalMethod(‘峨眉’,‘武当’)(‘少林’)();
mentalMethod(‘峨眉’,‘武当’,‘少林’)();

args包含了所有累积的参数
第一次调用：mentalMethod(‘困难’)→ args = [‘困难’]
第二次调用：fn(‘焦虑’)→ 合并后新的 args = [‘困难’, ‘焦虑’]
第三次调用：fn(‘压力’)→ 合并后新的 args = [‘困难’, ‘焦虑’, ‘压力’]
newArgs只是当前调用传入的新参数
在最终无参数调用时：fn()→ newArgs = []
如果使用 newArgs.join(‘,’)，结果会是空字符串：战胜

计时器

**首次启动：**外部代码 t = setTimeout(time, 1000);在1秒后第一次调用 time()。
**循环保持：**每次 time()执行时：
先 clearTimeout(t)清除当前定时器（避免潜在堆积）。
更新页面时间显示。
在函数末尾再次设置新的定时器 t = setTimeout(time, 1000);，等待1秒后重新触发 time()。
这个过程每秒重复一次，形成持续更新的时钟。

//compose.js
这是 函数式编程中的组合（compose）模式，在 中间件（middleware）​ 或 责任链（chain of responsibility）​ 设计模式中常见。主要用于将多个函数按顺序组合成一个流水线，每个函数处理数据后传递给下一个。这里是一个异步串行的中间件组合器实现。
核心机制
函数柯里化：compose接收多个中间件函数，返回一个新的组合函数
控制反转：每个中间件接收 (value, next)两个参数，必须调用 next()才能继续执行后续中间件
流水线处理：数据依次经过每个中间件处理

完善 js/compose.js 中 compose 函数的 TODO 部分，完成以下目标：
compose 函数接收多个中间件函数作为参数，每个函数中必须调用 next(arg) 将函数的执行值传递到下一个参数。
所有中间件函数接受两个参数：initialValue 和 callback。initialValue 是将通过中间件函数传递的初始值，而 callback 是在所有中间件函数执行完毕后将被调用并传入最终值的函数。
compose 函数的执行逻辑如下：它按照传入的顺序执行中间件函数，对每个中间件函数传递当前值。如果中间件函数返回一个值，则通过 next(arg) 的像是将其传递给下一个中间件函数。
compose 函数返回一个新的中间件函数，可以用于按顺序执行合并后的中间件函数。
compose 执行示例如下：

function add(str, next) {
str+=‘2’
next(str);
}
function add2(str, next) {
str+=‘3’
next(str);
}
const processdemo = compose(add,add2); //传入的函数会按照顺序执行
processdemo(“1”, (finalValue) => {
console.log(finalValue) // 123
});
const processdemo2 = compose(add2,add);
processdemo2(“1”, (finalValue) => {
console.log(finalValue) // 132
});

框架中间件（如 Express、Koa）的核心机制
洋葱模型：函数按顺序执行，层层传递
控制反转：每个函数不直接调用下一个，而是通过 next()回调
异步友好：虽然这里是同步，但模式支持异步操作
可插拔：可以动态添加/移除中间件

封装一个支持异步的管道函数

封装一个支持异步的 pipeline 管道函数，能够按顺序执行一系列异步操作，每个异步操作的结果将作为下一个异步操作的输入。
需要简单数据转换链（如数据处理、ETL）
函数参数说明：

  • initialValue：管道的初始值（即 sequence 中第一个函数的参数）。它是整个异步管道的起点。第一个异步步骤将以此值开始，并且后续步骤将在前一步骤的输出基础上进行。
  • sequence：是一个由具有返回值和可以传参的函数组成的数组，函数可以是普通函数也可以是 Promise 函数。每个函数接收前一个步骤的输出（即该函数的参数是上一个函数的执行结果），并返回一个 Promise。这个数组定义了整个管道中的处理步骤和它们的顺序。
    函数返回值说明：
  • pipeline 函数返回一个 Promise，这个 Promise 最终解析为整个管道执行完成后的结果。

实现选项卡动态切换功能

display: none
优点：元素完全从渲染树中移除，不参与布局、绘制和合成
性能影响：最小，浏览器完全忽略这个元素
重新渲染：切换时会引起重排（reflow）
z-index
优点：只是改变堆叠顺序，元素本身仍存在于渲染树中
性能影响：
如果只改变 z-index，通常只引起重绘（repaint），比重排快
但如果触发了新的层创建，会引起合成（composite）

元素数量：
大量元素：使用 display: none节省内存
少量元素：z-index切换更快
切换频率：
频繁切换：opacity或 visibility更好
偶尔切换：display也可以
是否需要保持布局：
需要占位：visibility: hidden或 opacity: 0
不需要占位：display: none

/* 推荐：需要频繁切换的隐藏/显示 /
.toggle-element {

visibility: hidden;
transition: opacity 0.3s;
}

visibility: visible;
}

/* 推荐：不常切换的完全隐藏 /

display: none; / 默认隐藏 */
}

.modal.active {

z-index: 1000; / 合理设置层级 */
}
