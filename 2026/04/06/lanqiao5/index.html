地址识别

他发现只要将自己的详细信息粘贴进识别框，识别框就能将相关信息自动提取并完成填写
请完善 js/index.js 中 parseContactInfo 函数的 TODO 部分代码，实现以下目标：
在地址输入框下方的文本域内（id=“info”）输入数据后，点击识别按钮（id=“transferBtn”）时，对输入的数据进行提取。parseContactInfo 函数接收一个字符串，为信息输入框（id=“info”）输入的数据（只会包含姓名、电话、地址三部分信息，各部分信息之间以空格分隔），要返回一个对象，对象中包含提取的信息，例如输入的数据为：
15277775555 重庆市渝中区解放碑步行街99号 刘十二
返回的数据为：
{
name: “刘十二”,
phone: 15277775555,
address: “重庆市渝中区解放碑步行街99号”
}

  • name 表示联系人姓名。
  • phone 表示联系人电话。
  • address 表示联系人地址。
    每个信息的提取规则如下：
  • name：2-4 位的中文字符。
  • phone：11 位的数字。
  • address：中文字符开头且由中文、英文、数字的组成的至少 4 位的字符。
    提示：
  • 匹配中文的正则为：[\u4e00-\u9fa5]。
    需要注意的是，输入的各部分信息的顺序可能是不一致的，例如输入的数据为：
    周八 15277775555 西湖7号，
    或者
    西湖7号 15012345678 周八
    返回的数据也为：
    {
    name: “周八”,
    phone: 15012345678,
    address: “西湖7号”
    }
    trim()于移除字符串两端的空白字符。

解密

你有一个加密的压缩包，需要暴力破解，已知压缩包密码是一组数字组成的数组，这组数字从左到右依次增大。现在需要完成一个暴力解密工具，可以帮我们返回所有可能的密码。
完善 js/index.js 中的 getPossiblePasswords 函数，该函数接受两个参数：

  • max：密码数组中的最大数字
  • count：密码数组的长度
    该方法返回值为一个二维数组，里面包含所有的可能解密的数组。解密的数组从1开始，无需考虑二维数组中包含的数组顺序以及密码数组中数字的顺序。即[[2,4],[3,4]]跟[[4,3],[4,2]] 是一样的。
    示例一：
    假设 max = 4、count = 2，函数则返回可能解密的数组为
    [
    [2,4],
    [3,4],
    [2,3],
    [1,2],
    [1,3],
    [1,4],
    ]
    示例二：
    假设 max = 1、count = 1，函数则返回可能解密的数组为
    [
    [1]
    ]
    假设压缩包的密码为 [4,9,18,24] 。当输入最大值大于或等于密码数组中的最大值，并且数组长度正确。则可得到解密密码。完成后效果如下图所示：

搜索重试

在这个项目中，我们探索了一种搜索结果加载的过程，并引入了重试机制来处理加载失败的情况。通过模拟异步搜索操作，我们在页面中展示了搜索结果，并在搜索失败时提供了重试选项。这个项目旨在提供一种优雅而可靠的方式来处理搜索操作中的错误。
补充 js/index.js 中 retry 函数的 TODO 部分，retry 允许一个 promise 函数在操作失败时进行多次重试，直到操作成功或达到最大重试次数为止。
该函数接收三个参数， 参数的作用和解释如下：

  • operation 为一个函数，该函数会返回成功或者失败的 promise。
  • maxAttempts 最大重试次数，类型为数字 number。
  • delayBetweenRetries 重试之间的时间间隔（单位：ms），类型为数字 number。
    如果异步操作（即执行 operation() ）在规定次数内返回成功，retry 函数会返回 operation() 执行成功时的 resolve 值；如果异步操作返回失败，并且达到了最大重试次数，函数会抛出一个错误 throw error，error为 operation() 执行失败时返回的 reject 值。
